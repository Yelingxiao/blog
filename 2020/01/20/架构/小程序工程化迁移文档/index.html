小程序工程化迁移文档

发表于 2020-01-20 | 分类于 前端工程化

码云代码仓库地址：https://gitee.com/f2ee/miniProgramCli

目录结构

迁移步骤

  1. 下载miniProgramCli项目到本地，将miniProgramCli除了node_modules,src和project.config.json，.git(注意.git文件)全部复制到目标项目。新建src文件夹将项目内原有文件全部放入src文件夹内，除了project.config.json和.git文件，他们在外层目录下。

  2. 在project.config.json加上一行配置“miniprogramRoot”: “dist/“，目的是微信小程序编辑器编译文件指向dist/目录下。（支付宝没有project.config.json，可以忽略这个步骤，编辑器手动指向目标项目的根目录下）。

  3. 支付宝小程序需要修改(微信小程序不需要这个步骤)。

    • webpack.config.js 俩处output 配置下 globalObject: ‘wx’ —> globalObject: ‘my’ , stylesLoaders配置下 name: ‘path.wxss’ —>name: ‘path.acss’

    • scripts —> auto_ptp —> helper —> config.js 修改 埋点事件列表

  1. yarn install 或 npm install 。

  2. yarn lint 或 npm run lint (node >= 9.10.0)。

  3. yarn dev 或 npm run dev 手动修改eslint 报错，最好将模块规范修改为es6 module规范。

  4. 在微信开发者工具 —> 详情 —> 本地设置 勾掉es6 转es5, 增强编译， 勾起自定义出来命令，上传前预处理加上（默认为： npm run build-prod）,可自定义修改。

  5. yarn dev 或 npm run dev 命令行没有报错和编辑器控制台没有报错即为迁移完成。

开发文档

  1. yarn install 或 npm install 开发前确保自己的依赖包下载完全
  2. npm run np xxx (如npm run np demo, 在pages内新增一个自定义模版文件，并在app.json pages下添加了对应路由)
  3. npm run dev 或 npm run test 或 npm run pro 来切换需要对应开发的环境
  4. Eslint 推荐使用ESLint VScode 插件可智能提醒哪里有lint错误，VScode setting 勾选上☑️Eslint: Auto Fix On Save 可自动修复部分eslint 错误。
  5. 建议关闭 文件自动保存
  6. 支持less语法
  7. 支持npm引入第三方包（如：npm install loadsh）
  8. 支持别名
    • @ === src
    • src === src
    • utils === src/utils
    • components === src/component
  9. npm run upImg 可将src/images内所有文件自动上传cdn
  10. 在less和js文件内写 “~images/xxx.png”。打包出图片路径可自动替换为cdn链接
  11. npm run replaceImg 替换源码内图片路径为cdn链接
  12. npm run ptp 自动埋点
  13. 微信开发这工具点击上传 自动跑 npm run build-prod的命令 并自动加上查找并加上埋点
